# 工作规范

## 项目定位

古典著作自动化分析管线 — 用 Omega 纯数学理论分析经典古代著作，自动生成内容并发布到社交网络。

## 环境与工具链

- Python: `python3`，长时间运行每 20s 输出一次进度
- 工作语言默认中文，对外发布内容中英双语
- n8n 工作流以 JSON 格式存储在 `n8n/workflows/`

## 核心原则

1. **数学严谨性** — 所有分析基于 Omega 形式化结果，引用须可追溯到 automath Lean 4 证明
2. **自动化优先** — 全流程无人工干预，从文本输入到社交发布一键完成
3. **可复现** — 每次分析结果可由脚本重新生成，参数和版本齐备
4. **内容质量** — 生成的内容须经过 LLM 自审查，避免 AI slop

## 目录约定

- `texts/` — 古典文本语料库（纯文本 + 元数据）
- `analysis/` — Omega 数学分析模块
- `content/` — LLM 内容生成
- `publish/` — 社交网络发布适配器
- `n8n/` — n8n 自动化工作流

## 与 automath 的关系

本项目是 automath 的下游消费者：
- 通过 `discovery-export` 获取 Lean 4 定理清单
- 通过 `notebooklm-pipeline` 生成多媒体内容
- 数学分析结果须引用具体的 Omega 定理编号和证明路径
